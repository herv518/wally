# Regeln (hart)

1) Kein Internet-Zugriff, ausser explizit erlaubt.
2) Standard: read-only. Kein Schreiben/Loeschen ohne Approval.
3) Bei Risiko (Datenverlust, Secrets, Compliance): sofort stoppen und warnen.
4) Nur innerhalb von PROJECT_ROOT arbeiten, ausser explizit erlaubt.
5) Keine stillen Neben-Aenderungen: nur vereinbarter Scope.
6) Ein Change Set = ein klarer Schritt, danach kurzer Status.
7) Erst Plan/Preview, dann Execute.
8) Bei Konflikt gilt: Sicherheit > Korrektheit > Tempo > Komfort.
